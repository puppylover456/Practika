# Инструкция по запуску проекта

## Способ 1: Через Visual Studio 2022

### Шаг 1: Открыть проект
1. Запустите Visual Studio 2022
2. Выберите **File → Open → Project/Solution**
3. Найдите папку `учебка` и выберите файл **CraftStore.csproj**
4. Нажмите **Open**

### Шаг 2: Скомпилировать проект
1. Нажмите **Build → Build Solution** (или `Ctrl+Shift+B`)
2. Убедитесь, что в окне **Output** нет ошибок компиляции
3. Если есть ошибки - проверьте, что все файлы на месте

### Шаг 3: Запустить проект
1. Нажмите **Debug → Start Debugging** (или `F5`)
2. Или нажмите **Debug → Start Without Debugging** (или `Ctrl+F5`)
3. Откроется консольное окно с меню магазина

## Способ 2: Через терминал VS Code

### Шаг 1: Открыть проект в VS Code
1. Запустите **Visual Studio Code**
2. Выберите **File → Open Folder...**
3. Найдите и откройте папку `C:\Users\adeli\Desktop\учебка`
4. VS Code автоматически определит проект .NET

### Шаг 2: Открыть терминал
1. Нажмите **Terminal → New Terminal** (или `Ctrl+`` ` - обратная кавычка)
2. Или используйте меню **View → Terminal**
3. Внизу откроется встроенный терминал (PowerShell или Command Prompt)

### Шаг 3: Восстановить зависимости (если нужно)
В терминале выполните:
```powershell
dotnet restore
```

### Шаг 4: Скомпилировать проект
```powershell
dotnet build
```

### Шаг 5: Запустить проект
```powershell
dotnet run
```

**Альтернативный способ:**
- Можно использовать кнопку **▶ Run** в правом верхнем углу VS Code (если установлено расширение C#)
- Или нажмите `F5` для запуска с отладкой

## Тестирование функций

После запуска вы увидите главное меню:

```
=== МАГАЗИН РУКОДЕЛИЯ 'МАСТЕРСКАЯ' ===
1. Каталог материалов
2. Наборы для творчества
3. Консультация по проекту
4. Оформить заказ
5. Мастер-классы и советы
6. Статистика мастерской
7. Поиск клиента
8. Выход
```

### Рекомендуемый порядок тестирования:

1. **Каталог материалов** (пункт 1)
   - Проверьте, что материалы отображаются по категориям
   - Убедитесь, что видна полная информация (цена, остаток, вид творчества)

2. **Наборы для творчества** (пункт 2)
   - Проверьте отображение наборов по сложности
   - Убедитесь, что показывается стоимость и доступность

3. **Консультация по проекту** (пункт 3)
   - Зарегистрируйте нового клиента (введите ФИО, телефон, email)
   - Выберите тип проекта (например, "шитье")
   - Проверьте рекомендации материалов и наборов
   - Создайте проект

4. **Оформить заказ** (пункт 4)
   - Найдите клиента по email или телефону
   - Выберите или создайте проект
   - Добавьте материалы (введите ID материала, затем количество)
   - Проверьте расчёт стоимости
   - Завершите заказ (введите 0)

5. **Статистика мастерской** (пункт 6)
   - Проверьте отображение выручки
   - Убедитесь, что показываются клиенты и популярные виды творчества

6. **Поиск клиента** (пункт 7)
   - Найдите клиента по email или телефону
   - Проверьте отображение информации о клиенте

## Возможные проблемы

### Ошибка компиляции: "The type or namespace name 'X' could not be found"
- **Решение:** Убедитесь, что все файлы (.cs) находятся в одной папке
- Проверьте, что все файлы имеют правильный namespace `CraftStore`

### Ошибка: "Program does not contain a static 'Main' method"
- **Решение:** Убедитесь, что файл `Program.cs` существует и содержит метод `Main`

### Проект не запускается
- **Решение:** 
  1. Закройте все консольные окна
  2. Выполните `dotnet clean`
  3. Выполните `dotnet build` заново
  4. Запустите `dotnet run`

## Установка .NET SDK

### Проверка установки .NET

Откройте терминал (PowerShell или Command Prompt) и выполните:
```powershell
dotnet --version
```

Если вы видите ошибку **"dotnet" не является внутренней или внешней командой**, значит .NET SDK не установлен.

### Вариант 1: Установка через Visual Studio 2022 (рекомендуется)

Если у вас установлена **Visual Studio 2022**, она уже включает .NET SDK:
1. Откройте Visual Studio Installer
2. Найдите Visual Studio 2022 в списке
3. Нажмите **Modify** (Изменить)
4. Убедитесь, что установлен компонент **".NET desktop development"** или **"ASP.NET and web development"**
5. Если нет - установите его и нажмите **Modify**

После установки перезапустите терминал и проверьте:
```powershell
dotnet --version
```

### Вариант 2: Установка только .NET SDK (без Visual Studio)

1. Перейдите на официальный сайт: https://dotnet.microsoft.com/download
2. Выберите **.NET 6.0 SDK** (или новее)
3. Скачайте установщик для Windows (x64)
4. Запустите установщик и следуйте инструкциям
5. **Важно:** После установки **перезапустите терминал** (закройте и откройте заново)

### Проверка после установки

После установки откройте **новый** терминал и выполните:
```powershell
dotnet --version
```

Должна отобразиться версия, например: `6.0.xxx` или `8.0.xxx`

### Если команда всё ещё не работает

1. **Перезагрузите компьютер** (иногда требуется для обновления PATH)
2. Или добавьте .NET в PATH вручную:
   - Обычно .NET устанавливается в: `C:\Program Files\dotnet\`
   - Добавьте этот путь в переменную окружения PATH
